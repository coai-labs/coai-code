# 安全政策

> English: [SECURITY.md](./SECURITY.md)

## 支持的版本

我们为以下版本的 CoAI Code 提供安全补丁：

| 版本  | 支持状态           |
| ----- | ------------------ |
| 0.1.x | :white_check_mark: |

不再支持更早的预发布版本。请在报告安全问题前升级到最新版本。

## 提交漏洞报告

**请不要在 GitHub 上公开提交 issue 来报告安全漏洞。**

请发送邮件至 **chonebz@gmail.com**，并在邮件中提供：

- 漏洞描述及其潜在影响
- 复现步骤或概念验证（如有）
- 受影响的版本
- 建议的修复方案（如有）

### 响应时间表

| 时间节点   | 处理动作                                               |
| ---------- | ------------------------------------------------------ |
| 48 小时内  | 确认收到报告                                           |
| 7 天内     | 初步评估和严重性评级                                   |
| 30–90 天   | 补丁开发、测试和协调披露（根据严重性决定具体时间）     |

我们遵循负责任披露原则。修复就绪后，我们将：

1. 准备补丁版本。
2. 私下通知报告者以便其验证修复效果。
3. 同步发布 GitHub Security Advisory 和新版本。
4. 对报告者予以致谢（除非其希望匿名）。

我们请求报告者在协调披露日期之前保守漏洞细节。

## 范围

本政策覆盖 `coai` 二进制文件及其发布的库代码。第三方依赖不在范围内，此类问题请向相应的上游维护者报告。

## 不在范围内

以下情况**不**视为本政策覆盖的安全漏洞：

- 需要物理访问用户设备才能触发的问题
- 针对项目贡献者的社会工程学攻击
- 非 `coai` / `coai-code` 所致的终端用户环境漏洞
